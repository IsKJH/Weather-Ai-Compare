01

실험 개요

같은 날씨 앱 명세를 AI는 Android Kotlin/Compose MVVM으로, 직접 구현은 Flutter/Dart Clean Architecture로 진행합니다. 언어와 아키텍처는 다르지만, 구현 기능과 화면 명세는 동일합니다.

비교 초점
각 Phase를 완성하는 데 실제로 걸리는 시간 — AI 생성 시간과 손 코딩 시간을 Phase별로 누적 비교합니다.
언어 차이 참고
Flutter는 프로젝트 세팅(pubspec, build_runner)이 Android보다 약간 더 걸립니다. 단순 숫자 비교 시 이 점을 감안해야 합니다.
Phase 구성
V1(기본 날씨) → V2(시간별 예보·즐겨찾기·새로고침) → V3(자유 스마트 기능 1개) 3단계로 진행합니다.

  • MVVM: ViewModel + UiState + Composable 직결
  • StateFlow로 단방향 데이터 흐름
  • 별도 의존성 없음 (표준 Compose만)
  • 구조는 단순, 코드가 적고 빠르게 생성 가능
Human 선택: Clean Architecture
도메인/데이터/프레젠테이션 레이어 분리. 규모가 커질 때 유지보수 유리하지만 초기 보일러플레이트가 많아 Phase 1 구현 시간이 늘어났습니다.
AI 선택: MVVM
명세에서 MVVM을 명시적으로 요구했기 때문에 AI는 보일러플레이트 없이 핵심 기능만 빠르게 구현했습니다. 파일이 적고 코드량도 적습니다.
시간 차이의 주요 원인
① 프레임워크 전환(Flutter 세팅) ② 아키텍처 레이어 설계 시간 ③ 타이핑 속도 vs AI 출력 속도. AI는 3가지 모두 없거나 매우 짧습니다.
